React源码阅读笔记-commit阶段

发表于 2023-03-14 分类于 web前端 阅读次数： Valine：

在初次创建渲染那篇文章的最后一个方法performConcurrentWorkOnRoot中可以看到，当执行完render阶段后，会返回一个exitStatus，表示render阶段是否全部执行完毕，如果成功执行完毕，则接下来调用finishConcurrentRender方法，进入commit阶段
commit阶段的核心方法为commitRoot，该方法中主要分为三个部分：

  • before-mutation阶段
  • mutation阶段（执行DOM操作）
  • layout阶段

入口函数

commit阶段的的核心方法为commitRootImpl

before-mutation 之前

在正式进入commit阶段之前，会先触发useEffect执行后返回的回调方法（unmount），然后重置属性，并以普通优先级异步调用useEffect的方法
如果root有更新标识或子树上有更新标识，则进入commit阶段，否则直接将root.current指向新fiber树

layout之后

  • 对useEffect的处理

  • 再次调用ensureRootIsScheduled方法，检测是否有新的任务需要调度

  • 执行同步任务

    before-mutation阶段

  • 处理focus状态

  • 遍历fiber树，执行class组件的getSnapshotBeforeUpdate函数

  • 将根节点下的dom节点移除

    mutation阶段

    根据不同类型，进行不同的DOM节点操作，但核心都会执行两个方法，分别为

  • recursivelyTraverseMutationEffects：深度遍历执行删除操作

  • commitReconciliationEffects：根据是否有placement标识进行DOM节点的挂载操作

    核心方法-recursivelyTraverseMutationEffects

    在render阶段的beginWork方法中，在创建fiber树，老fiber节点标记删除时，会将需要删除的fiber节点存放在父fiber的deletions属性中
    该方法会遍历父节点的deletions属性，对删除列表中的每一个fiber节点执行删除操作，之后，会根据子树标识上是否有其他标识，来进行对子树的深度遍历

    对fiber节点执行删除操作，根据节点的不同类型，做不同的处理方式，如函数组件，则执行destroy方法，类组件则执行componentWillUnmount方法

  • 如果是原生组件，则清除ref

  • 如果是原生文本组件或原生组件，则遍历子fiber节点，调用commitDeletionEffectsOnFiber方法，向下遍历，然后在最上层的父节点，调用DOM的removeChild方法，移除老DOM节点

  • 如果是函数组件，则依次执行useInsertionEffect和useLayoutEffect的destroy方法，并向下遍历节点

  • 如果是类组件，则执行componentWillUnmount方法，并向下遍历节点

    核心方法-commitReconciliationEffects

    如果fiber中有placement标识，则表示有插入逻辑，调用commitPlacement方法。执行结束后，需要将该标识置空。
    该方法核心是进行DOM的挂载操作，根据是否有兄弟节点，执行DOM的原生方法insertBefore或appendChild

  • 如果父fiber是根节点类型，如果有兄弟节点，则在根DOM节点上调用insertBefore插入，否则调用appendChild方法，将所有子节点挂载到根DOM节点上

  • 如果父fiber是原生组件类型

    • 如果父fiber上有ContentReset标识，则文本重置
    • 如果有兄弟DOM节点，则使用DOM的insertBefore方法，插入DOM节点，否则使用DOM的appendChild方法，挂载子节点们 DOM节点的操作顺序：删除操作是从根节点开始由上到下，挂载跟新操作时从叶子节点由下到上

      函数组件-FunctionComponent

  • 调用commitReconciliationEffects执行DOM节点挂载操作

  • 如果存在Update标识，

    • 执行useInsertionEffect的销毁函数
    • 执行useInsertionEffect的回调函数，并收集此次的销毁函数
    • 执行useLayoutEffect的销毁函数

      类组件-ClassComponent

  • 调用commitReconciliationEffects执行DOM节点挂载操作

  • 如果是update阶段且存在ref，需要将ref进行置空

    原生组件-HostComponent

  1. 如果有placement标识，则进行DOM挂载操作
  2. 如果有ref标识，且update阶段，则将ref置空
  3. 如果有contentReset标识，则进行DOM的内容重置操作
  4. 如果有update标识，且存在老DOM节点，则调用commitUpdate方法进行DOM的更新操作

    updatePayload存放的是需要更新设置的DOM属性，在render阶段的completeWork方法中会以[key1, value1, key2, value2…]的形式存放在fiber的updateQueue上

layout阶段

layout阶段的入口函数，设置了循环的起始指针nextEffect，调用核心函数commitLayoutEffects_begin

layout阶段开始函数

  • 从subtreeRoot开始遍历，判断subtreeFlags是否有标识

  • 如果有则将nextEffect指向子节点

  • 如果没有标识则调用commitLayoutMountEffects_complete

    横向遍历节点，判断节点flags上是否有LayoutMask标识

  • 如果有则调用commitLayoutEffectOnFiber函数

  • 然后将nextEffect指向兄弟节点

  • 如果不存在兄弟节点，则将nextEffect执行父节点

    针对fiber进行layout阶段的处理

  • 如果是函数组件，则执行useLayoutEffect回调函数

  • 如果是类组件，mount阶段则执行componentDidMount函数，update阶段则执行componentDidUpdate函数，最后调用commitUpdateQueue函数，处理update回调函数，如this.setState({}, callback)中的callback

  • 如果是原生根组件，则会调用commitUpdateQueue函数

  • 如果是原生组件，在mount阶段且存在更新标识，则会调用commitMount函数，处理autoFocus，以及img标签的src赋值

  • 最后，处理ref的赋值

    总结： layout阶段，主要是深度遍历，由下至上对节点的生命周期的调用，以及副作用的回调的执行，和ref赋值
