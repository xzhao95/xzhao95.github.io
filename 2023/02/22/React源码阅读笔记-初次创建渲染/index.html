阶段一：创建根节点

创建一个react应用从ReactDOM.createRoot(container)开始，返回一个root，这个root可以用来渲染一个react节点并挂载到DOM上。
这个createRoot方法，根据调用，最终方法在react/packages/react-dom/src/client/ReactDOMRoot.js文件里

在createRoot方法中，主要执行了createContainer方法，新建了一个FiberRootNode，并返回一个ReactDOMRoot，它的唯一属性_internalRoot指向FiberRootNode

在createFiberRoot方法中，主要创建了FiberRootNode对象，以及初始化了一个空的FiberNode，并设置这两个对象的连接

总结

所以createRoot做了如下事情

  1. 创建了FiberRootNode以及FiberNode，将container放在FiberRootNode中
  2. 返回一个ReactDOMRoot，render方法也在这个root上

阶段二：ReactDOMRoot.prototype.render

创建完ReactDOMRoot后，调用该root上的render方法root.render(<App/>)
在ReactDOMRoot.prototype.render方法中，核心方法为updateContainer方法

updateContainer最后几个方法的调用，是该方法的核心，也就是update的入列处理以及update的调度更新
查看class组件的setState方法可知，在setState方法中，最后也是执行enqueueUpdate以及scheduleUpdateOnFiber函数。

从React API中，可查看到React.Component的初始化方法，setState方法主要执行了this.updater.enqueueSetState(this, partialState, callback, ‘setState’);
而updater的赋值是在render阶段针对class组件的处理中，也就是在constructClassInstance函数中的adoptClassInstance中，对class instance进行赋值操作

在updateContainer方法中主要做了：

  1. 创建了一个update
  2. 在enqueueUpdate中，update入列处理
    1. 将update.next指向update自己本身，形成一个环状链表
    2. 将fiber.updateQueue.shared添加concurrentQueues（并发队列）中
    3. 将fiber.updateQueue.shared.interleaved指向update
    4. 返回FiberNodeRoot
  3. update的调度更新

enqueueUpdate：update的入列处理

scheduleUpdateOnFiber：update的调度更新

在该函数中，主要是对一些优先级的赋值，以及针对update的标记，核心函数是ensureRootIsScheduled，判断是否存在新的任务需要调度

ensureRootIsScheduled中，主要做的事情：

  1. 检查是否有超时任务，获取当前任务的优先级，判断是否有任务需要执行，没有任务的话，重置callback并返回
  2. 检查是否有正在执行的任务，并比较一下两个任务的优先级，如果相等的话，则标识正在执行的任务优先级比较高，直接返回，否则，暂停正在执行的任务，优先执行当前任务
  3. 根据当前任务的优先级，判断是同步执行还是异步执行
  4. 如果是异步执行，则异步执行performConcurrentWorkOnRoot方法，在concurrent模式下，注册任务，并加入到任务队列中
  5. 最后，将更高优先级的任务，赋值到root.callbackNode上

    scheduleCallback函数主要是注册调度任务，相关代码在schedule包中

render和commit阶段的入口

performConcurrentWorkOnRoot和performSyncWorkOnRoot是render和commit阶段的入口函数。

  1. 如果存在阻塞任务或该任务已超时，则调用renderRootSync进行同步渲染，否则调用renderRootConcurrent进行异步渲染，这两个方法是render阶段的入口函数，主要用于构造fiber树
  2. render阶段执行完毕后会返回退出状态值，根据不同状态值做不同处理，如果状态值为正常完成结束，则调用finishConcurrentRender方法（commit阶段的入口函数）进入commit阶段，该阶段主要是挂载DOM节点
  3. 最后会重新调用ensureRootIsScheduled方法，判断是否有新任务需要重新发起调度
  4. 如果root.callbackNode没有置空（在commit阶段，会将callbackNode置空），则表示由于当前帧时间不够，渲染被阻断，需要返回当然函数，重新渲染
