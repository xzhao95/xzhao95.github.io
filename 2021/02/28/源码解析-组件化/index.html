源码解析--组件化

发表于 2021-02-28 分类于 web前端 阅读次数： Valine：

一、组件挂载

1. createComponent

在mountComponet中执行render方法，进入到createElement方法里，之前逻辑跟普通节点挂载都是一样的。
在createElement中当判断tag是个对象或tag不是普通的html标签，则直接调用createComponent方法

2. 构造子类的构造函数

在createComponent方法里，context.$options._base是Vue，在初始化initGlobalApi中对Vue.options._base有进行赋值操作。之后执行Vue.extend方法进行构造子类的构造函数。

在Vue.extend方法里，核心内容就是使用原型链的方法，继承Vue，构造一个Vue的子类Sub。然后再该子类Sub添加扩展属性，以及一些全局api等。最后缓存该子类，防止重复创建。在之后new一个Sub实例时，就会执行this._init方法，再次走Vue实例初始化的逻辑

3. 安装组件钩子函数

installComponentHooks方法主要将componentVNodeHooks里的钩子函数合并到data.hooks里。方便在组件patch过程中做一些额外的操作。

4. 实例化VNode

根据各类参数实例化一个VNode。该VNode没有children。createComponent最后返回该VNode。

5. patch

render生成虚拟节点树后，执行update方法进行patch。
在patch方法里执行createElm方法，在该方法里执行createComponent，如果不是个组件，则返回false

在createComponent方法里，判断是否存在之前安装的钩子函数，钩子函数里是否有init方法。如果存在，则执行init钩子函数。在init方法里会对vnode.componentInstance进行赋值，否则为空。

在init方法里，如果不是已缓存组件，则在走else部分。执行createComponentInstanceForVnode方法，创建一个Vue实例

在createComponentInstanceForVnode函数里构造一个内部组件的参数，定义_isComponent为true，_parentVnode为组件的虚拟节点，parent为当前组件实例。最后vnode.componentOptions.Ctro就是之前的构造器Sub。在实例化的过程中，执行了this._init方法。

在_init方法里根据之前赋值的_isComponent判断组件，执行initInternalComponent函数。在该函数里，主要进行的操作在于把之前createComponentInstanceForVnode中设置的几个参数合并到内部$options上。因为不存在el，所以不走this.$mount方法

回到init钩子函数中，完成子组件实例化后，调用组件的$mount方法，创建子组件的渲染wacher后，调用子组件的render方法，创建子组件的虚拟节点树。在_render方法里，_parentVnode为父虚拟节点，vnode是组件的渲染虚拟节点，将vnode.parent指向_parentVnode，建立该组件vnode的父子关系。

之后调用子组件的_update方法，也就是patch方法，进行vnode渲染。vm._vnode指向的是render返回的渲染vnode，而在_render方法里，vm.$vnode指向的父vnode。

其中使用setActiveInstance方法，用来控制当前上下文Vue实例。setActiveInstance使用了闭包的方法，缓存了上一次的activeInstance实例，待该组件实例完成_patch和_update方法后，activeInstance又会重新指向它的父实例。

在patch里，oldVnode为空，直接走createElm方法。如果组件中都是普通标签，则走之前普通标签的逻辑，先创建一个父节点占位符，然后递归调用createElm创建子节点。构建完整的组件节点树。

在组件执行createElm时没有传入parentElm，所以在createComponent方法里，在组件树创建完成递归返回后执行insert方法进行DOM节点插入。所以是DOM节点的插入是先子后父。

该笔记为Vue.js技术揭秘课程的学习笔记，源码复盘记录。
